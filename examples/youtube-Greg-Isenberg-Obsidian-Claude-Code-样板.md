## Claude Code 到底是什么

Vin 先讲基础。Claude Code 是一个跑在命令行里的 agent，可以用自然语言控制你的电脑。

> 我可以直接说"在我桌面上创建一个文件，内容是 Hello Greg"，它就去做了。以前我得打开文本编辑器，手动建文件。现在我直接说一句话，文件就躺在桌面上了。

但这还不是重点。重点是**上下文（context）**。

当你把一个 agent 当做真能帮你做事的伙伴用，你会发现你总在反复解释同一件事——这个项目是什么、上次我们讨论到哪、我为什么这么想。每次新 session 都要重来。

> 很多人在网页上用 Claude 或 ChatGPT，它们有"memory"功能，但你没法控制那里面装了什么。你不知道它记得什么、不记得什么。

解决方式很简单：**把你的项目描述、想法、长期偏好写成文件，放在桌面或 vault 里。然后在每次对话里 reference 那个文件**。Claude Code 能读你的文件，所以你不用反复口头解释。

> 整个游戏就是把好的上下文喂给这只野兽。
>
> ——Greg

很多人说 Claude Code 用起来"没那么 gamechanging"。Vin 的诊断是：**他们没在对的时间喂对的上下文**。

## Obsidian 到底是什么

它是一个跑在 markdown 文件之上的界面。你的 vault（保险库）就是一堆 markdown 文件的文件夹，Obsidian 只是一个更好看、更智能的阅读/编辑层。

Vin 演示了他自己的 vault：

- 每日笔记（daily notes）
- 学到东西就建的笔记（比如"我从某某人身上学到了什么"）
- 项目文档
- 人物档案（比如他给 Greg Eisenberg 专门建了一个文件）

Obsidian 和普通文件夹最大的区别是它允许你**在文件之间建立双向链接（backlinks）**。

> 今天我上 Greg Eisenberg 的播客。我写这个笔记的时候把"Greg Eisenberg"这个链接加进去——现在这个笔记就连到了我之前关于 Greg 的所有笔记。超有意思。

他打开了 graph view，圆圈密密麻麻，每个圆圈是一个文件，连线是文件之间的链接。"Personal Agent Infrastructure"这个笔记连到了 Agentic AI、Telegram、Toby（Shopify 创始人）、Presence Log、Claudebot、他自己的播客"The Other Stuff"等等。

> 大家之所以喜欢 Obsidian，是因为这种相互连接的能力。它更像你大脑的工作方式——大脑一直在连接各种模式。

但 Greg 提了一个关键问题：**"我看到为什么它好玩了，但这怎么让我得到更好的输出？"**

## Obsidian CLI：让 Claude Code 读懂整个 vault

Obsidian 最近发布了一个叫 Obsidian CLI 的工具。它能让 Claude Code 不仅读你 vault 里的文件，还能**读懂文件之间的互相连接**。

Vin 认为这是整个组合真正厉害的地方：

> Claude Code 能看到：哦，这个文件连到那个文件，那个文件又连到另外三个文件。然后它能开始在你自己看不出来的地方发现模式——也许你一年里一直在写关于同一个模式的东西，但你自己没意识到。AI 可以直接告诉你："嘿，你有没有发现你在这几个不同领域的笔记里一直在绕同一个话题？"

他的原话：

> 这种"第一次看清自己一直在想什么"的灯泡时刻，会让你的理解力、观点、正在做的事情都出现巨大的跃迁。

## Vin 的自定义 slash 命令

这是整期视频最实用的部分。Vin 在 Claude Code 里自定义了一堆 slash 命令，每个命令触发一段专门的流程。他现场演示了这些：

### 日常操作类

**/context** — 加载完整上下文。读取 context 文件 + 每日笔记 + 跟着 backlink 建立你当前生活和工作的完整画面。

> 我打开新 session 的时候，就先打 /context。它会自动读一大堆关于我当前状态的文件。
> 新 session 一开始就知道我在做什么、最近在想什么、哪些是重要项目。省去所有解释。

**/today** — 晨间回顾。拉取日历、任务、iMessage、过去一周的每日笔记，组织成今天的优先级计划。

> 为什么有意义？你当然可以给 agent 访问你日历和任务的权限。但它没有"你在想什么、为什么"这层上下文。如果我最近每天在写关于某个技术或项目的笔记，日历里应该对得上才对。有这层上下文，agent 能更准地建议你该做什么、该接什么会议。

**/close-day** — 一天结束处理。提取待办、浮现 vault 里的新连接、检查那些"我还没有确定把握"的假设的置信度是否需要更新。

> 我有一堆假设，每个都有一个置信评分。这条我感觉很稳，那条我不太确定。/close-day 帮我每天刷新这些评分。

### 思考伙伴类（Vin 最爱用的）

**/ghost** — 以我自己的语气回答一个问题。它先从 vault 里建一个我的"语气档案"，然后用我的语气写答案，最后还会评价自己模仿得像不像。

**/challenge** — 压测当前信念。用 vault 里自己过去的历史记录找矛盾点、反证据、思维转变。

> 为什么重要？如果我想持续成长，就要保证自己的观点不是过度偏见或者过于狭窄的。/challenge 专门挑战我。

**/emerge** — 浮现 vault 里**暗示但从未明说**的想法。把散落的前提拼成结论，命名那些我自己都没名字的模式，指出没被表达过的方向。

> 超级有用。我可能在 vault 里绕着一个想法转了好几年，什么都没说透。有人或者 AI 简单点出来："嘿，你一直在绕这个。"——这就是巨大的突破。

**/drift** — 对比我**说过想做什么** vs **过去 30-60 天实际在做什么**。指出我在回避什么、在逃避什么。

> 听这期播客的观众会喜欢这条。

**/trace** — 追踪一个想法在 vault 里是怎么演化的。

Vin 现场运行了 `/trace 我和 Obsidian 的关系`，AI 回了一份完整的演化史：

- **2024 年 12 月之前**：Obsidian 完全不在画面里。他写过一篇长文《How I Take Notes in the West End of Toronto》，那时候他的系统是 Mac Whisper 语音 + LLM 对话 + Canopia 空间映射 + 实体本子 + Fuseland + Are.na
- **2025 年 1-5 月**：发现 Obsidian 但很怀疑——"双向链接可能没那么有用？"
- **关键转折**：某天笔记里他突然意识到："最重要的不是给通用标签打链接（播客、健身、电影制作），而是为我的每一个**模式/理论/项目/观点**建单独的笔记，然后从那些笔记往外链。"
- **2026 年 1 月：爆发期**
- 当前：前沿已经不是 Obsidian 本身，而是**如何让 agent 自动执行任务**——vault 和 agent 之间的边界

他说：

> 我觉得绝了——我只是在记笔记，AI 就能穿越我生活所有领域告诉我这些。就算作为一个父亲，我也能反思我在学什么。一台电脑能对我知道这么多、还能浮现出这些模式——太疯了。

**/connect** — 把两个不同领域的笔记连起来。

Vin 演示了 `/connect 电影制作 世界观构建`。AI 跑过他 vault 里所有电影制作相关笔记 + 所有世界观构建相关笔记，然后给出"桥接点 1/2/3"，每个桥接点都引自他自己写过的原文。

### 建造类

**/schedule** — 要不要接这个会？

他演示了问 AI："我今天下午 2 点能接 Greg 的会吗？"AI 查了他日历，又读了他每日笔记，回答：

> 你今天已经排满了：上午录 Greg 的播客，接着团队午餐，再和 Peter + Vince 开会。你 2 月 17 日的笔记显示 Greg 那期已经一直挂在心上。vault 里有专门给 Greg Eisenberg 的笔记。建议：2 点不要加，而且可能都不需要另外开会。

Greg 的反应：

> 这其实是对的答案。

**/ideas** — 综合项目建议生成器。跑过整个 vault 结构 + 最近每日笔记 + 每一个项目 context 文件，花大约 5 分钟生成一份报告。

Vin 展示的报告里面居然包含：
- **该建的工具**：比如"/graduate"命令（把每日笔记里零散的想法提升为独立笔记）
- **要读的系统级建议**：Christopher Alexander 的 pattern language 应用到数字空间
- **该见的人**：按他最近在想的东西，建议联系某某做某件事的人
- **该做的播客选题**
- **vault 本身的结构改进建议**（比如哪些文件是"孤儿"没被链到、哪些 link 是 unresolved）

### 关键领悟：让 AI 决定你该用什么命令

Vin 分享了一个反直觉的方法：

> 一开始我是自己想命令。但后来我回退一步问 AI："基于你从我 vault 里读到的一切，你判断我现在的理解水平和项目阶段，**建议一些能把我从当前水平带到更高水平的命令**。"

AI 就提议了 /graduate 这种他自己都没想过但确实需要的命令。

Vin 总结这个思路：

> 让 AI 去更高的抽象层面帮你想，而不是在具体命令层面上。

## 上下文文件（Context Files）怎么写

Vin 给每个项目都写了专属的 context 文件。他犹豫着打开了其中一个"The Other Stuff Working Context"（他自己的播客节目），说：

> 这很私人，但你看——"最近发生了什么变化"、"核心信念：研究是基础"、"最好的对话感觉像发现"、"团队成员"——所有这些信息都在里面。所以 AI 完全知道我播客团队有谁、我最近在探索什么假设。

另一个叫"Personal Workflow Context"的文件写了他**每日作息、做事偏好、不喜欢怎么工作**。还有一个"Personal Agent Infrastructure"记录他**如何思考委派给 agent 这件事的演化**。

这一切带来一个关键结果：**所有这些文件都会在 /context 命令执行时被自动加载进当前对话**。

## 关于 OpenClaw 的讨论

Greg 追问了一个关键问题：**这套东西和 OpenClaw 怎么协同？**

OpenClaw 是可以不需要你每次提示就自动做事的 agent 框架。它能读 vault、找连接、以你的名义做决策。

Vin 的思路：

> OpenClaw 可以跟我用 /schedule 一样，自己读我的 vault、找连接、帮我做决策。我不再是在管理一个 agent 或和另一个人类讨论事情——我只是专注于管理这个 vault。Vault 成了新的源头。

> 我只管不断让 vault 更完整、更新、更深——这样不管任何 agent 来 query 它，它都能拿到我最新的状态。如果 agent 做了错误决定，我就去改 vault，而不是去"调教"某个具体 agent。

但 Vin 也诚实谈到了风险：

> 如果 Obsidian 真是你的"第二大脑"，给 OpenClaw 这种自主 agent 访问权限——说实话，挺吓人的。

> 我故意给了 Claude Code 和各种 agent 很多访问权限，因为我想搞清楚这些东西到底是什么、人和电脑的关系正在怎么变。但隐私是一个真问题。连我做这个演示，每个命令都要做 demo 版本，以免播出时暴露太多个人信息。

## 反思作为一种生活方式

Greg 问了一个直击痛点的问题：

> 很多人从一个会到下一个会，我们忙，我们当父母。但我感觉人越长大，写和反思越来越少。你怎么把反思塞进生活里？

Vin 给了两个答案：

**一、情感角度**：

> 能回看自己过去的笔记，意识到"哦我这个人是在持续变化的"——这是生命中特别棒的一部分。反思自己怎么变化、世界怎么变化、技能怎么变化。

**二、功能角度**：

> 我写笔记多是因为**写作就是我产生想法的方式**。当我坐下来打字写东西，想法才来。这件事你光说是想不出来的，写着写着就内化了。我喜欢有好想法、喜欢在进步——所以我写。

然后他讲了对当下 LLM 时代最重要的一点：

> 写作现在还是**你如何向 agent 委派任务**的基础。这是一个全新维度。如果你能培养写作习惯，你就有大量的上下文可以传给 agent——这会极大地扩大你能委派的事、能建的东西。

## 关于私人 vault 里放什么

Greg 追问了一个务实问题：**会议记录呢？你不参加的会议的记录呢？比如"Tommy 和 Vince 开了个会，我想把记录放进来"？**

Vin 的回答是：

> 你想放什么都行。Granola/Gemini Notes 的会议转录放进来都可以。关键是你怎么组织——你可以建一个"meetings/"目录，每次做完会就把 Granola 笔记扔进去。如果你打标签比如"#podcast"，agent 就会发现这些文件跟你播客是相关的。

但 Vin 自己有一条关键规则：

> **我不让 agent 往 vault 里写文件。**

> 我可以轻易让它写，但我不想。因为我想控制 vault 里的一切——我想让它只 pull "我怎么想"，不要 pull "它怎么想"。如果它开始自己建文件，那它找的模式是自己写的还是我写的？边界就乱了。我定了一条"严格分离"的规则——它只在对话里输出，然后我自己决定什么该写进 vault。

这是一个非常值得抄的原则。

## 这一期的三个关键迁移点

### 1. 把 vault 当作你的单一事实源

不要跟 agent 谈。跟你的 vault 谈。**每次 agent 出错，你不是去调教 agent，你是去改 vault**。这一条从根本上改变了人机协作的拓扑结构——从"对话循环"变成"源头维护"。

### 2. 让 AI 从更高抽象层建议命令

反直觉但特别有效：不要自己想命令，**让 AI 看了你的整个 vault 之后告诉你它觉得你该加什么命令**。这是"元自我提升"——AI 在帮你**提升你提升自己的方式**。

### 3. 你的写作习惯就是你未来生产力的上限

> 你能委派给 agent 多少事，取决于你能给它多少上下文。你的上下文就是你写下来的东西。

这对那些想用 AI 放大个人产出的人是一个新框架：**写得越多、越诚实，agent 能帮你的事就越多**。

## 可迁移启示（写给啸天本人）

1. **Vin 的 slash 命令体系值得你抄**——你已经有 OpenClaw 跑在 Obsidian vault 上，差的是把"思考伙伴类"命令（/challenge, /emerge, /drift, /trace）用你的项目语境做一套。这跟你"营销 AI 化"主线契合度极高——你的 SCRM 产品经理经验、业务主线、新潮传媒交付记录、查理编剧课笔记都在 vault 里，Claude Code 现在已经能读全，关键是**把你要问的那些问题沉淀成命令**。

2. **"不让 agent 写 vault 里的文件"这条规则你应该抄**——你用 OpenClaw 自动化一堆事，但 vault 里的内容必须全部是你亲手写的。这样 OpenClaw 找模式时，找的永远是"啸天怎么想"，不是"OpenClaw 怎么想"。边界必须清楚。

3. **"/trace 这个想法是怎么演化的"是你想做的"学习复盘"的金钥匙**——你的查理编剧课整理、91 集字幕重排、OnceLab 战略演化、甚至"营销 AI 化"这个主线本身，都应该能被 /trace 出来。这是一种**对自己思想历程的考古**。

4. **"Context Files"和你的 `.agents/product-marketing-context.md` 是一个思路**——Vin 给每个项目、每个身份（播客主、团队 lead、父亲）都写了 context 文件。你可以给自己所有项目（新潮、老丁、MBTI、抖音 AI Lab、公众号）都建一个。

5. **OnceLab 未来的 "Personal Agent Infrastructure" 可以学这个**——Vin 这套模式是**可产品化的**。你的 OnceLab 给企业交付"Agent 基础设施"时，完全可以卖这套"vault + CLI + slash commands"的 methodology。你的差异化是比 Vin 更懂中文场景、更懂国内企业工作流。
